ErlPort and Python: Opaque Objects in LFE

In our last post in this series, we touched upon calling Python module hierarchies from LFE. In this session we’re going to revisit that, but with a twist: what happens when ErlPort doesn’t know how to serialize that data?

Warming Up

Same drill as the last post; we’re going to re-use the demo repo. If you need to get set up from scratch, just do this:

$ git clone git@github.com:oubiwann/erlport-demo.git
$ cd erlport-demo/python
$ python3.4 -m venv .venv
$ . .venv/bin/activate
$ cd ../lfe
$ make repl

If you’ve already downloaded and run the REPL before, you can skip the build step with this:

$ make repl-no-deps

Let’s do our walkthrough, starting with creating two separate running Python servers:

> (set `#(ok ,pid-1) (python:start))
#(ok <0.32.0>)
> (set `#(ok ,pid-2) (python:start))
#(ok <0.33.0>)

Opaque Objects

When (python:call ...) returns data that doesn’t have an explicit match defined in ErlPort, it is encoded as an “opque object”. This can be demonstrated with something as simple as a Python dictionary object:

> (python:call pid-1 'builtins 'dict '(((a 1) (b 2))))
#($erlport.opaque python
  #B(128 2 125 113 0 40 99 101 114 108 112 111 114 116 46 101 ...))
> (set (= `#($erlport.opaque python ,data-2) dict)
       (python:call pid-1 'builtins 'dict '(((a 1) (b 2)))))
#($erlport.opaque python
  #B(128 2 125 113 0 40 99 101 114 108 112 111 114 116 46 101 ...))

Let’s try that again, but this time calling the now() function in Python’s datetime.datetime:

> (python:call pid-1 'datetime 'datetime.now '())
#($erlport.opaque python
  #B(128 2 99 100 97 116 101 116 105 109 101 10 100 97 116 101 116 105 ...))
>

Let’s set that to some variables so we can examine more closely. In particular, let’s capture the whole things in the now variable, and capture the binary data in the data variable:

> (set (= `#($erlport.opaque python ,data) now)
       (python:call pid-1 'datetime 'datetime.now '()))
#($erlport.opaque python
  #B(128 2 99 100 97 116 101 116 105 109 101 10 100 97 116 101 116 105 ...))
